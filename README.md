# data-warehouse-project
This project demonstrates the development of a data warehouse using MySQL. It integrates data from multiple sources through the ETL (Extract, Transform, Load) process and organizes it into structured tables for efficient storage, querying, and analysis to support reporting and data-driven decision making.

## 🚀 Project Requirements

### Data Warehouse Development (Data Engineering)

#### Objective

The main goal of this project is to build a modern **data warehouse using SQL** that integrates sales data from different systems. The warehouse will support analytical queries and help generate meaningful insights for better business decision-making.

#### Specifications

* **Data Sources:** Load datasets from two systems (ERP and CRM) provided as CSV files.
* **Data Cleaning:** Identify and fix data quality problems before storing the data.
* **Data Integration:** Merge the datasets into a unified and structured data model optimized for analysis.
* **Project Scope:** Only the most recent dataset will be used; maintaining historical records is not required.
* **Documentation:** Provide clear documentation of the data model for both business users and analytics teams.

---

### Business Intelligence & Analytics

#### Goal

Create SQL-based analytical queries to generate insights related to:

* **Customer behavior and patterns**
* **Product performance**
* **Sales trends**

These insights help stakeholders understand key business metrics and support strategic planning and decision-making.
